<h1 id="overview">Overview</h1>
<p>Your task is to decode a qr code.
You get the qr code as 2 dimensional array, filled with numbers. 1 is for a black field and 0 for a white field.
It is always a qr code of version 1 (21*21), it is always using mask 0 ((x+y)%2), it is always using byte mode and it always has error correction level H (up to 30%). The qr code won't be positioned wrong and there won't be any squares for positioning exept the three big ones in the corners.</p>
<p>You should return the message inside the qr code as string.<br>The QR Code will always be valid and be of <code>version 1</code>, meaning the decoded message will never be more than 8 characters long. The way to decode a complete QR-code is explained below, but keep in mind that in this kata, you'll only have to decode the parts in green in the picture below:</p>
<img border="0" alt="dcode-image-v3-v2" src="https://i.ibb.co/7RjZbXG/dcode-image-v3-v2.jpg">

<h1 id="decoding-a-qr-code">Decoding a QR-code</h1>
<p>Here comes the explaination on how to decode a qr code. You can skip it, if you already know how it works:</p>
<h3 id="positioning-information">Positioning information</h3>
<p>First of all we have to take a look at the big three positioning fields in the corners.</p>
<img border="0" alt="qrmatrixpkf2" src="https://i.ibb.co/99YtbCj/qrmatrixpkf2.jpg">

<p>You can see on the image that these fields are connected.<br>The fields are just there for the positioning and I told before that the qr code will be always positioned correctly so you can ignore them.</p>
<h3 id="mask-information">Mask information</h3>
<p>The bits next to the positioning fields give us information about the mask and the error correction level the qr code uses. I wrote above that it is always mask 0 and always error correction level H, so we can also ignore that stuff.</p>
<img border="0" alt="qrmatrixti2" src="https://i.ibb.co/7jzjQy5/qrmatrixti2.jpg">

<h3 id="reading-information">Reading information</h3>
<p>Now we start with the important stuff. Go through the qr code like the red arrow shows on this picture (btw I made it with paint so don't judge me)</p>
<ul>
<li>We start in the lower right corner</li>
<li>Then we go one to the left</li>
<li>Then we go one up and one to the right</li>
<li>And so on just look the red arrow</li>
</ul>
<p><em><strong>Important:</strong></em> Remember that everything inside the blue boxes has another purpose than encoding information, so don't forget to skip those parts.</p>
<img border="0" alt="dcode-image-v3" src="https://i.ibb.co/zsqr5JZ/dcode-image-v3.png">

<p>In the image below, you may find the complete pattern to read information in a QR-code. But keep in mind you'll be handling only "QR-code version 1", so you don't need to read the full set of data (see <a href="https://i.ibb.co/7RjZbXG/dcode-image-v3-v2.jpg" data-turbolinks="false" target="_blank">picture at the top</a> if needed).</p>
<img border="0" alt="1920px-QR_Character_Placement.svg" src="https://i.ibb.co/rFk5Dc5/1920px-QR-Character-Placement-svg.png">

<h3 id="decoding-information">Decoding information</h3>
<p>We have to build a bit sequence now. In order to do that we will use mask 0 definition which is <code>((x+y)%2)==0</code>, where:</p>
<ul>
<li>x and y are the indexes of our 2 dimensional array (0-based)</li>
<li>if the condition of our mask is true, we have to convert the pixel: black -&gt; 0 and white -&gt; 1</li>
<li>A mask is used to prevent long sequences of identical bits so that a scanner can better recognize the code</li>
</ul>
<p>For each black field add 1 to our bit sequence and for each white field add 0 to our bit sequence, don't forget that many bits get converted because of mask 0.</p>
<p>Let's do the first bits together:</p>
<ul>
<li>We start with the first pixel (in the lower right corner, where also the red arrow begins) which is black, but we have to use mask because (20+20)%2 is 0, therefore we don't add 1 to our bit sequence but 0.</li>
<li>Next field is white. This time we don't use mask because (20+19)%2 isn't 0, so we add a 0 to our bit sequence.</li>
<li>Next field is black. This time we don't use mask because (19+20)%2 isn't 0, so we add a 1 to our bit sequence.</li>
</ul>
<p>Important (!): Since we're limiting ourselves to version 1, we have to continue that process only until our bit sequence is 76 long, because the input will never be longer than eight characters.</p>
<p>At the end you get a bit sequence:</p>
<pre><code>bits:      0100000000100100100001101001000011101100000100011110110000010001111011001111
legend:    MMMMssssssss...

-   "M": mode bits (4 bits)
-   "s": size message bits (8 bits)
-   ...: message bits and error correction information
</code></pre>
<p>This bit sequence is representing the following information</p>
<ul>
<li>First 4 bits show mode: <code>0100</code>. This isn't important for us, because I told you before that we will use always byte mode in this kata.</li>
<li>The following 8 bits show the length of the encoded word: <code>00000010</code>. This is the binary representation of number 2, so we know our word is 2 characters long.</li>
<li>The following bits are data bits followed by error correction bits (but you can ignore error correction bits, because there won't be any errors). We know our word is 2 chars long, so we take the next 16 bits (because 1 char=8 bits):<ul>
<li>First group of 8 bits: <code>01001000</code>. This is <code>72</code> in decimal which is ascii value of <code>"H"</code>.</li>
<li>Second group of 8 bits: <code>01101001</code>. This is <code>105</code> in decimal which is ascii value of <code>"i"</code>.</li>
</ul>
</li>
</ul>
<p>Since we don't handle in this kata error correction, we got our word/output: <code>"Hi"</code>.</p>
<p>Good luck :)</p>
