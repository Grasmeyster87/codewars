<div class="markdown prose max-w-5xl mx-auto overflow-x-auto break-words" id="description"><p>Consider a sequence <code>u</code> where u is defined as follows:</p>
<ol>
<li>The number <code>u(0) = 1</code> is the first one in <code>u</code>.</li>
<li>For each <code>x</code> in <code>u</code>, then <code>y = 2 * x + 1</code> and <code>z = 3 * x + 1</code> must be in <code>u</code> too.</li>
<li>There are no other numbers in <code>u</code>.</li>
</ol>
<p>Ex: 
<code>u = [1, 3, 4, 7, 9, 10, 13, 15, 19, 21, 22, 27, ...]</code></p>
<p>1 gives 3 and 4, then 3 gives 7 and 10, 4 gives 9 and 13, then 7 gives 15 and 22 and so on...</p>
<h4 id="task">Task:</h4>
<p>Given parameter <code>n</code> the function <code>dbl_linear</code> (or dblLinear...) returns the element <code>u(n)</code> of 
the ordered (with &lt;) sequence <code>u</code> (so, there are no duplicates).</p>
<h4 id="example">Example:</h4>
<p><code>dbl_linear(10) should return 22</code></p>

<div class="markdown prose max-w-5xl mx-auto overflow-x-auto break-words" id="description"><p>Розглянемо послідовність <code>u</code>, де u визначено наступним чином:</p>
<ol>
<li>Число <code>u(0) = 1</code> є першим у <code>u</code>.</li>
<li>Для кожного <code>x</code> у <code>u</code>, тоді <code>y = 2 * x + 1</code> та <code>z = 3 * x + 1</code> також повинні бути в <code>u</code>.</li>
<li>У <code>u</code> немає інших чисел.</li>
</ol>
<p>Наприклад:
<code>u = [1, 3, 4, 7, 9, 10, 13, 15, 19, 21, 22, 27, ...]</code></p>
<p>1 дає 3 та 4, потім 3 дає 7 та 10, 4 дає 9 та 13, потім 7 дає 15 та 22 і так далі...</p>
<h4 id="task">Завдання:</h4>
<p>Завдяки параметру <code>n</code> функція <code>dbl_linear</code> (або dblLinear...) повертає елемент <code>u(n)</code> упорядкованої (з &lt;) послідовності <code>u</code> (тому дублікатів немає).</p>
<h4 id="example">Приклад:</h4>
<p><code>dbl_linear(10) має повернути 22</code></p>
<h4 id="note">Примітка:</h4>
<p>Зосередьте увагу на ефективності</p>
</div>

Щоб зрозуміти логіку розв’язання цього завдання, давай розкладемо його по кроках:

### 1. Визначення послідовності
- Початковий елемент: \(u(0) = 1\).
- Для кожного елемента \(x\) у послідовності додаються:
  - \(y = 2x + 1\)
  - \(z = 3x + 1\).
- Послідовність впорядковується за зростанням і не містить дублікатів.

### 2. Проблема прямого підходу
Якщо просто генерувати всі можливі значення, а потім сортувати, то:
- буде багато зайвих обчислень,
- з’являться дублікати,
- сортування стане дорогим при великих \(n\).

Тому потрібен більш ефективний метод.

### 3. Ефективна стратегія
Можна використати **два вказівники (індекси)**:
- один для формули \(2x+1\),
- інший для формули \(3x+1\).

#### Алгоритм:
1. Починаємо з масиву \(u = [1]\).
2. Заводимо два індекси: \(i2 = 0\), \(i3 = 0\).
3. На кожному кроці:
   - Обчислюємо кандидати:
     - \(next2 = 2 \cdot u[i2] + 1\),
     - \(next3 = 3 \cdot u[i3] + 1\).
   - Беремо мінімальне з них як наступний елемент.
   - Якщо мінімальне дорівнює \(next2\), збільшуємо \(i2\).
   - Якщо мінімальне дорівнює \(next3\), збільшуємо \(i3\).
   - Додаємо мінімальне в послідовність, якщо його ще немає (щоб уникнути дублікатів).
4. Повторюємо, поки не отримаємо \(n\)-й елемент.

### 4. Приклад
Для \(n = 10\):
- \(u = [1, 3, 4, 7, 9, 10, 13, 15, 19, 21, 22]\).
- Отже, \(u(10) = 22\).

### 5. Ключова ідея
Цей метод працює подібно до алгоритму злиття двох впорядкованих послідовностей:
- одна генерується через \(2x+1\),
- інша через \(3x+1\).
Ми поступово будуємо єдину впорядковану послідовність без дублювання.


### Як працює цей код:
- Масив `u` зберігає побудовану послідовність.
- Індекси `i2` та `i3` вказують на позиції, з яких генеруємо наступні кандидати.
- На кожному кроці обчислюємо два можливих значення (`next2`, `next3`), беремо мінімальне і додаємо його в послідовність.
- Якщо мінімальне значення співпадає з обома кандидатами, збільшуємо обидва індекси.
- Перевірка на дублікати гарантує, що послідовність містить лише унікальні числа.

Цей підхід працює ефективно навіть для великих `n`, бо ми не генеруємо зайві числа і не сортуємо весь масив.
